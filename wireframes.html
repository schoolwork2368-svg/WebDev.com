Wireframes

A wireframe is a low-fidelity, simplified outline of a webpage or application. It serves as a blueprint that defines the structure, layout, information architecture, and user flow.

The key purpose of a wireframe is to focus on the placement of elements and functionality without being distracted by colors, fonts, or images. It's a critical step in the design process that happens before any visual design or coding begins.

Goals of Wireframing

  • Define the hierarchy of information on the page.
  • Clarify how the user will navigate and interact with the site.
  • Provide a clear structure for the development team to build upon.
  • Allow for quick and cheap iteration on design ideas.

Common Tools

Wireframes can be drawn by hand, but digital tools are more common for collaboration. Popular tools include Figma, Sketch, Adobe XD, and Balsamiq.

  • Balsamiq: Known for its sketch-like wireframes.
  • Figma: A collaborative design tool that supports wireframing.
  • Adobe XD: A powerful tool for designing and prototyping user experiences.
  • Sketch: A vector-based design tool popular among UI/UX designers.
